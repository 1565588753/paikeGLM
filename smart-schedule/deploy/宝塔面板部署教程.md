# 智能排课系统 - 宝塔面板图形化部署教程

> 全程在宝塔面板Web界面操作，无需敲命令行

---

## 前提条件

- 已安装宝塔面板 11.x
- 服务器：CentOS 7+/Ubuntu 20+/Debian 10+
- 最低配置：2核4G内存，40G硬盘
- 推荐配置：4核8G内存，80G硬盘

---

## 第一步：安装基础软件

### 1.1 安装 Nginx

1. 登录宝塔面板
2. 点击左侧菜单 **「软件商店」**
3. 搜索 **「Nginx」**，选择 **1.24** 版本
4. 点击 **「安装」**，选择「编译安装」，等待安装完成

### 1.2 安装 MySQL 5.7

1. 在软件商店搜索 **「MySQL」**
2. 选择 **5.7** 版本，点击 **「安装」**
3. 安装完成后，点击 MySQL 右侧的 **「设置」**
4. 在「配置修改」中确认 `max_connections` 至少为 500
5. 点击 **「保存并重启」**

### 1.3 安装 Python 项目管理器

1. 在软件商店搜索 **「Python项目管理器」**
2. 点击 **「安装」**，等待安装完成
3. 安装完成后，点击 **「设置」** 进入管理界面
4. 点击 **「版本管理」**，安装 **Python 3.10** 版本

### 1.4 安装 Redis（可选但推荐）

1. 在软件商店搜索 **「Redis」**
2. 选择最新版本，点击 **「安装」**
3. 安装完成后保持默认配置即可

## 第二步：创建数据库

### 2.1 添加数据库

1. 点击左侧菜单 **「数据库」**
2. 点击 **「添加数据库」**
3. 填写信息：
   - **数据库名**：`smart_schedule`
   - **用户名**：`smart_schedule`
   - **密码**：点击「生成密码」生成一个强密码，**请记下这个密码**
   - **访问权限**：选择「本地服务器」
4. 点击 **「提交」**

### 2.2 导入数据库结构

1. 在数据库列表中，找到 `smart_schedule` 数据库
2. 点击右侧的 **「导入」** 按钮
3. 点击 **「上传文件」**，依次上传以下两个文件：
   - `sql/init_schema.sql`（数据库表结构）
   - `sql/init_data.sql`（示例数据）
4. 上传完成后，先选中 `init_schema.sql`，点击 **「导入」**
5. 导入成功后，再选中 `init_data.sql`，点击 **「导入」**
6. 导入完成后，点击右侧的 **「管理」** → 进入 phpMyAdmin
7. 在左侧选择 `smart_schedule` 数据库，确认有 21 张表

## 第三步：上传项目文件

### 3.1 创建网站目录

1. 点击左侧菜单 **「文件」**
2. 进入 `/www/wwwroot/` 目录
3. 点击 **「新建目录」**，输入 `smart-schedule`
4. 进入 `smart-schedule` 目录
5. 再创建以下子目录：
   - `backend`
   - `frontend`
   - `backups`
   - `logs`

### 3.2 上传后端代码

1. 在本地电脑上，将 `backend/` 目录下的所有文件打包为 `backend.zip`
2. 在宝塔文件管理器中，进入 `/www/wwwroot/smart-schedule/backend/`
3. 点击 **「上传」**，上传 `backend.zip`
4. 上传完成后，右键点击 `backend.zip` → **「解压」** → 解压到当前目录
5. 解压后删除 `backend.zip`
6. 确认目录结构如下：
   ```
   /www/wwwroot/smart-schedule/backend/
   ├── app/
   │   ├── __init__.py
   │   ├── main.py
   │   ├── config.py
   │   ├── database.py
   │   ├── models/
   │   ├── schemas/
   │   ├── api/
   │   ├── services/
   │   ├── engine/
   │   └── utils/
   ├── requirements.txt
   └── start.sh
   ```

### 3.3 上传前端代码

1. 在本地电脑上，将 `frontend/dist/` 目录下的所有文件打包为 `frontend-dist.zip`
2. 在宝塔文件管理器中，进入 `/www/wwwroot/smart-schedule/frontend/`
3. 点击 **「上传」**，上传 `frontend-dist.zip`
4. 右键点击解压到当前目录
5. 确认目录结构如下：
   ```
   /www/wwwroot/smart-schedule/frontend/
   ├── index.html
   ├── favicon.svg
   └── assets/
       ├── index-xxx.js
       ├── index-xxx.css
       └── ...（其他文件）
   ```

## 第四步：配置后端环境

### 4.1 创建 .env 配置文件

1. 在宝塔文件管理器中，进入 `/www/wwwroot/smart-schedule/backend/`
2. 点击 **「新建文件」**，文件名输入 `.env`
3. 右键点击 `.env` → **「编辑」**，填入以下内容：

```env
# 数据库配置（请将密码替换为第二步中记下的密码）
DATABASE_URL=mysql+pymysql://smart_schedule:你的数据库密码@localhost:3306/smart_schedule

# Redis配置
REDIS_URL=redis://localhost:6379/0

# 安全密钥（请随机生成一个32位以上的字符串）
SECRET_KEY=请替换为一个随机字符串例如abc123xyz789

# 备份目录
BACKUP_DIR=/www/wwwroot/smart-schedule/backups

# 日志目录
LOG_DIR=/www/wwwroot/smart-schedule/logs

# 运行环境
ENVIRONMENT=production
```

4. 将 `你的数据库密码` 替换为第二步记下的数据库密码
5. 将 `SECRET_KEY` 后面的值替换为一个随机字符串（越长越好）
6. 点击 **「保存」**

### 4.2 使用 Python 项目管理器部署后端

1. 点击左侧菜单 **「软件商店」** → 找到 **「Python项目管理器」** → 点击 **「设置」**
2. 点击 **「添加项目」**
3. 填写项目信息：
   - **项目名称**：`smart-schedule`
   - **Python版本**：选择 `Python 3.10.x`
   - **框架**：选择 `FastAPI`
   - **项目路径**：`/www/wwwroot/smart-schedule/backend`
   - **启动方式**：选择 `uvicorn`
   - **启动文件**：`app.main:app`
   - **端口**：`8000`
   - **运行用户**：`root`
   - **是否安装模块**：勾选 ✅
   - **模块文件**：`requirements.txt`
4. 点击 **「提交」**
5. 等待 Python 项目管理器自动安装依赖（可能需要 3-5 分钟）
6. 安装完成后，确认项目状态为 **「运行中」**

> **如果 Python 项目管理器不支持 FastAPI 框架选项**，请按以下替代方式操作：
>
> 1. 在 Python 项目管理器中，选择「其他」框架
> 2. 启动命令填写：`/www/server/pyporject_evn/smart-schedule/bin/python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4`
> 3. 或者参考第五步的「手动创建进程守护」方式

## 第五步：配置 Nginx 网站

### 5.1 创建网站

1. 点击左侧菜单 **「网站」**
2. 点击 **「添加站点」**
3. 填写信息：
   - **域名**：如果有域名填域名，没有域名填服务器IP地址
   - **根目录**：`/www/wwwroot/smart-schedule/frontend`
   - **FTP**：不创建
   - **数据库**：不创建（已在第二步创建）
   - **PHP版本**：选择「纯静态」
4. 点击 **「提交」**

### 5.2 配置 Nginx 反向代理

1. 在网站列表中，找到刚创建的网站
2. 点击网站名称右侧的 **「设置」**
3. 点击左侧 **「反向代理」**
4. 点击 **「添加反向代理」**
5. 填写信息：
   - **代理名称**：`backend-api`
   - **目标URL**：`http://127.0.0.1:8000`
   - **发送域名**：`$host`
6. 点击 **「提交」**

### 5.3 修改 Nginx 配置（关键步骤）

1. 在网站设置中，点击左侧 **「配置文件」**
2. **完全替换**为以下内容（请根据实际情况修改 `server_name`）：

```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    # 前端静态文件
    root /www/wwwroot/smart-schedule/frontend;
    index index.html;

    # Vue Router history 模式支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # 后端API反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 300s;
        proxy_connect_timeout 75s;
        proxy_send_timeout 300s;
        client_max_body_size 50m;
    }

    # WebSocket 实时通知代理
    location /ws/ {
        proxy_pass http://127.0.0.1:8000/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 86400s;
    }

    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }

    # 日志
    access_log /www/wwwlogs/smart-schedule.log;
    error_log /www/wwwlogs/smart-schedule.error.log;
}
```

3. 将 `你的域名或IP` 替换为实际的域名或服务器IP
4. 点击 **「保存」**
5. 在弹出的提示中点击 **「重启」** Nginx

1. 在浏览器中访问：`http://你的服务器IP/api/health`

1. 在浏览器中访问：`http://你的服务器IP`
2. 应该看到智能排课系统的登录页面

### 6.3 测试登录

   - 用户名：`admin`
   - 密码：`admin123`
2. 登录成功后进入系统首页

## 常见问题排查

### 问题1：访问页面显示 502 Bad Gateway

**原因**：后端服务未启动

**解决**：
1. 打开 Python 项目管理器
2. 找到 `smart-schedule` 项目
3. 点击 **「重启」**
4. 如果重启失败，查看日志排查原因

### 问题2：后端启动失败，提示数据库连接错误

**原因**：`.env` 文件中数据库密码不正确

**解决**：
1. 打开文件管理器
2. 编辑 `/www/wwwroot/smart-schedule/backend/.env`
3. 检查 `DATABASE_URL` 中的用户名和密码是否正确
4. 保存后重启后端服务

### 问题3：API 请求返回 404

**原因**：Nginx 反向代理配置不正确

**解决**：
1. 检查网站配置文件中 `/api/` 的 `proxy_pass` 是否指向 `http://127.0.0.1:8000`
2. 确认后端服务在 8000 端口运行
3. 重启 Nginx

### 问题4：Python 依赖安装失败

**原因**：缺少系统依赖或网络问题

**解决**：
1. 在宝塔终端中执行：
   ```bash
   yum install -y gcc python3-devel mysql-devel  # CentOS
   # 或
   apt install -y gcc python3-dev default-libmysqlclient-dev  # Ubuntu
   ```
2. 然后在 Python 项目管理器中重新安装模块

### 问题5：Python 项目管理器找不到 FastAPI 框架

**解决**：手动创建进程守护

1. 在软件商店搜索安装 **「Supervisor管理器」**
2. 打开 Supervisor管理器 → 点击 **「添加守护进程」**
3. 填写信息：
   - **名称**：`smart-schedule`
   - **启动命令**：`/www/server/pyporject_evn/smart-schedule_310/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4`
   - **运行目录**：`/www/wwwroot/smart-schedule/backend`
   - **启动用户**：`root`
4. 点击 **「提交」**
5. 确认进程状态为「运行中」

> **注意**：启动命令中的 Python 路径可能不同，请根据实际情况调整。
> 可以通过宝塔终端执行 `which python3.10` 查找 Python 路径。

### 问题6：换课通知不实时

**原因**：WebSocket 连接未正确代理

**解决**：
1. 检查 Nginx 配置中是否有 `/ws/` 的代理配置
2. 确认 `proxy_http_version 1.1` 和 `Upgrade` 头已设置
3. 重启 Nginx

## 日常维护

### 重启后端服务

- **方式一**：Python项目管理器 → 找到项目 → 点击「重启」
- **方式二**：Supervisor管理器 → 找到进程 → 点击「重启」

### 查看后端日志

1. 点击左侧菜单 **「文件」**
2. 进入 `/www/wwwroot/smart-schedule/logs/`
3. 查看日志文件

### 更新系统

1. 在本地电脑上重新构建前端：`npm run build`
2. 将新的 `dist/` 文件打包上传到 `/www/wwwroot/smart-schedule/frontend/`
3. 将新的后端代码打包上传到 `/www/wwwroot/smart-schedule/backend/`
4. 在 Python 项目管理器中重启项目
5. 如果数据库结构有变更，在 phpMyAdmin 中执行新的 SQL 脚本

### 数据库手动备份

1. 点击左侧菜单 **「数据库」**
2. 找到 `smart_schedule` 数据库
3. 点击右侧的 **「备份」** 按钮
4. 备份完成后可点击 **「下载」** 保存到本地

## 安全建议

1. **修改默认密码**：首次登录后立即修改 admin 密码
2. **限制访问IP**：在网站设置 → 「访问限制」中设置允许访问的IP
3. **开启防火墙**：宝塔面板 → 安全 → 只放行 80、443 端口
4. **定期更新**：保持系统和软件版本最新
5. **SSL加密**：务必配置 HTTPS 证书
6. **数据库安全**：数据库只允许本地访问，不要开放远程连接
